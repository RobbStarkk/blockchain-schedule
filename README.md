# Система расписания и учета успеваемости
Смарт-контракт для для ведения расписания занятий, учета успеваемости и посещаемости. 
Расписание хранится как таблица 15x6x7, то есть: 15 недель (обычный размер семестра), 6 учебных дней, 7 временных ячеек в день. Значение в таблице может быть 0 (нет занятия) или равнятся *id* предмета.
# Как использовать?
* Метод **addSubject** добавляет предмет в расписание. Аргументы:
	* *uint _id* - уникальный ID предмета;
	* *uint _nClasses* - число занятий, не может быть больше 15;
	* *uint _day* - в какой день проводится занятие;
	* *uint _time* - в какое время проводить занятие (временная ячейка).
* Метод **addStudent** добавляет студента по адресу (также нужно указать его *id*).
* Метод **addTeacher** добавляет преподавателя по адресу (также нужно указать его *id*).
* Метод **addTeacherToSubject** позволяет администратору припесать преподователя к предмету. 
* Метод **chooseSubject** позволяет студенту записаться на предмет по *id*.
* Метод **approveStudent** позволяет преподавателю подтвердить участие студента на занятии.
* Метод **markStudentToSubject** позволяет преподавателю отметить студента на определенном занятии *_mark = 1* если студент был на занятии и 0, если иначе.
* Метод **gradeStudentToSubject** позволяет выставить оценку от 1 до 5 студенту за определенное занятие.
* Метод **getScheduleByStudent** возвращает расписание студента (только те занятия, на которых он подтвержден) с *_weekStart* по *_weekEnd*.
* **getScheduleByTeacher** аналогично для преподавателя.
* **getGradesBySubject** возвращает все оценки за предмет в формате массива структур вида (адрес студента, массив оценок). Если оценка равна 0, значит она не выставлена.
